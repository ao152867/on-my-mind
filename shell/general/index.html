Shortcuts

This is for zsh if you do this in another shell like bash then there are different shortcuts.
e.g. Ctrl + s freezes what is displayed on the terminal until you press Ctrl + q. Which is bad because it just stops displaying what is going on any key presses will continue to work and when you unfreeze whatever you did will have happened.
http://unix.stackexchange.com/questions/12107/how-to-unfreeze-after-accidentally-pressing-ctrl-s-in-a-terminal
Ctrl + U in bash will clear the line before the cursor position but in ZSH it clears all of it. Both store it in the buffer later use. Like with Ctrl + y will print out what was cut out.

Shortcut Description
Ctrl + a Move cursor to beginning of line
Ctrl + s Forward search (good if you’ve gone too far in a reverse search)
Ctrl + d On an empty line: Exit
Over text: same as Del key (deletes text under the cursor)
Ctrl + g Seems like it clears the line no matter where on the line you are. But it actually fails
Ctrl + f Same as the right arrow key
Ctrl + b Same as the left arrow key

If you press the Shift key then the same that would happen in other programs happens in the terminal, like:

Shortcut Description
Ctrl + Shift + a Select all
